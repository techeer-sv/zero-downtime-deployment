# 어떻게 업데이트 될까?
stop-first
1. 버젼 1.0.0 으로 서버가 3개 돌아가고 있음
2. 버젼 1.0.0 서버 하나를 stop하고 지움
3. 버젼 1.0.1 으로 서버 1개 띄우고 health check
4. 버전 1.0.0 서버 하나를 stop하고 지움
5. 버젼 1.0.1 으로 서버 1개 띄우고 health check
6. 버전 1.0.0 서버 하나를 stop하고 지움
7. 버젼 1.0.1 으로 서버 1개 띄우고 health check
8. 최종적으로 3개 서버가 버젼 1.0.1이 됨

start-first
1. 버젼 1.0.0 으로 서버가 3개 돌아가고 있음
2. 버젼 1.0.1 서버를 1개 띄우고 health check
3. 버전 1.0.0 서버 하나를 stop하고 지움
4. 버젼 1.0.1 서버를 1개 띄우고 health check
5. 버전 1.0.0 서버 하나를 stop하고 지움
6. 버젼 1.0.1 서버를 1개 띄우고 health check
7. 버전 1.0.0 서버 하나를 stop하고 지움
8. 최종적으로 3개 서버가 버젼 1.0.1이 됨

# 배포 스크립트 작성
1. 배포 스크립트를 실행할때 버젼과 방식을 argument로 보낸다
   - 예시: `bash deploy.sh v1.0.1 stop-first`
   - 방식 생략시 기본값: `start-first`
2. 현재 돌아가고 있는 컨테이너 상태를 확인한다
   - 컨테이너가 없을 경우 `deploy-rolling.sh` 로 신버전 3개 초기
   - 구버전 컨테이너가 실행 중이면 방식에 따라 순차 교체 시작

### stop-first
1. 구버전 3개 실행 중
--- 1회차 ---
2. `remove-old.sh <container>` 로 구버전 1개 제거 → 구버전 2개
3. `deploy-extra.sh <tag> 0` 으로 신버전 1개 추가 → 구버전 2개 + 신버전 1개 (총 3개)
4. 신버전 헬스체크 / 실패시 즉시 중단 (구버전 2개 + 신버전 1개 상태 유지)
--- 2회차 ---
5. `remove-old.sh <container>` 로 구버전 1개 제거 → 구버전 1개 + 신버전 1개 (총 2개)
6. `deploy-extra.sh <tag> 0` 으로 신버전 1개 추가 → 구버전 1개 + 신버전 2개 (총 3개)
7. 신버전 헬스체크 / 실패시 즉시 중단 (구버전 1개 + 신버전 2개 상태 유지)
--- 3회차 ---
8. `remove-old.sh <container>` 로 구버전 1개 제거 → 신버전 2개
9. `deploy-extra.sh <tag> 0` 으로 신버전 1개 추가 → 신버전 3개
10. 신버전 헬스체크 / 실패시 즉시 중단 (신버전 2개 상태 유지)
--- 완료 ---
11. 신버전 3개 실행 중

### start-first
1. 구버전 3개 실행 중
--- 1회차 ---
2. `deploy-extra.sh <tag> 1` 로 신버전 1개 추가 → 구버전 3개 + 신버전 1개 (총 4개)
3. 신버전 헬스체크 / 실패시 신버전 종료 후 중단 → 구버전 3개로 복귀
4. `remove-old.sh <container>` 로 구버전 1개 제거 → 구버전 2개 + 신버전 1개 (총 3개)
--- 2회차 ---
5. `deploy-extra.sh <tag> 1` 로 신버전 1개 추가 → 구버전 2개 + 신버전 2개 (총 4개)
6. 신버전 헬스체크 / 실패시 신버전 종료 후 중단 → 구버전 2개 + 신버전 1개 유지
7. `remove-old.sh <container>` 로 구버전 1개 제거 → 구버전 1개 + 신버전 2개 (총 3개)
--- 3회차 ---
8. `deploy-extra.sh <tag> 1` 로 신버전 1개 추가 → 구버전 1개 + 신버전 3개 (총 4개)
9. 신버전 헬스체크 / 실패시 신버전 종료 후 중단 → 구버전 1개 + 신버전 2개 유지
10. `remove-old.sh <container>` 로 구버전 1개 제거 → 신버전 3개
--- 완료 ---
11. 신버전 3개 실행 중

3. 모든 교체 완료 후 신버전 3개가 실행 중인 상태가 됨
