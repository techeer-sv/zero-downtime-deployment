# 어떻게 업데이트 될까?
1. 버젼 1.0.0 일단 돌아가고 있음
2. 버젼 1.0.1로 올려야됨 
3. v1.0.1인 서버를 띄움
4. 검사
5. v1.0.1에 요청을 일부만 돌림
6. 검사
7. 괜찮으면 비율을 늘림
8. 문제가 생기면 새버전으로 가는 요청을 중단

# 카나리를 위해서 필요한 것
1. 요청 비율 정하기 
    - 라우터의 웨이트 비율 바꾸기 5% 20% 50% 100%
    - 비율 바꿔줄때 사이에 대기 2분
2. 새버전 검사 성공기준(6번)
3. 컴포즈파일
   - 구버전 3개
   - 5% 1개 20% 2개 50% 3개
   
# 배포 스크립트 작성                                                        
1. 배포 스크립트를 실행할때 실행할 버젼도 같이 argument로 보낸다  
   - 예시: bash deploy.sh v1.0.1
2. 현재 돌아가고있는게 a인지 b인지 확인한다
   - 둘다 없을경우 a로 초기 배포 (카나리 없이 바로 100%)
   - a가 돌아가고 있으면 b를 카나리로 배포
   - b가 돌아가고 있으면 a를 카나리로 배포
   - 둘다 실행중이면 오류 종료 (이전 배포가 덜 끝난 상태)
3. 카나리를 1개 기동하고 헬스체크
4. traefik dynamic 디렉토리의 canary-routers.yaml을 templates 폴더 파일로 교체하며 단계적으로 비율 조정
   - 각 단계마다 replica 수도 같이 늘린다
   - 각 단계마다 2분 bake 후 헬스체크
   - 헬스체크 실패시 안정 환경 100% 복구 후 카나리 종료 (롤백)
5. 100%까지 올라가면 구버전 컨테이너 종료
